
 Análisis de Retención de Usuarios – Cohort Analysis (SQL)

## 📌 Problema
Comprender el comportamiento de retención de usuarios en una plataforma de e-commerce, evaluando qué porcentaje de usuarios permanece activo después de su registro.

## 🎯 Objetivo
Analizar la retención de usuarios a lo largo del tiempo mediante cohortes y calcular métricas clave en distintos intervalos (D7, D14, D21, D28).

## ⚙️ Qué hice
- Construí consultas SQL utilizando CTEs (Common Table Expressions)
- Segmenté usuarios por cohortes basadas en la fecha de registro
- Analicé la actividad de usuarios en diferentes intervalos de tiempo
- Calculé métricas de retención mediante agregaciones y condiciones

## 📊 Resultados
- Cálculo de tasas de retención en distintos periodos (7, 14, 21 y 28 días)
- Identificación de patrones de comportamiento de usuarios en el tiempo
- Generación de insights para entender el engagement y la fidelización

## 🛠️ Tecnologías utilizadas
SQL | CTEs | Análisis de cohortes | Retención de usuarios | KPIs

## 📊 Dataset
Los datos utilizados son simulados y representan el comportamiento de usuarios en una plataforma de e-commerce, incluyendo fechas de registro, actividad y estado de uso.

## 📁 Estructura del proyecto
- `queries.sql` → consultas SQL del análisis

## 💡 Aprendizajes
Este proyecto fortaleció mi capacidad para trabajar con análisis de cohortes, medir retención de usuarios y aplicar SQL avanzado para resolver problemas de negocio.
